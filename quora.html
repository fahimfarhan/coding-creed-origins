I assume you are asking for "must-know" knowledge for algorithm programming contests (e.g., the ACM-ICPC, Topcoder SRMs, ...). I'm not so sure if every programmer should know some number theory knowledge.

I participated in the ACM-ICPC for 4 years, entering the World Finals twice. I am not really very strong at Number Theory problems, but I wish to answer as far as I could. The following items are categorized based on subjective judgement.

Basics:
  • LCM and GCD
  • (Extended) Euclidean algorithm
  • Modular arithmetic (addition, subtraction, multiplication)
  • Modular multiplicative inverse ("division") - Existence and computation
  • (Fast) Prime list generation
  • Fast prime factorization (with/without pre-process)
  • Exponentiation by squaring (e.g., computation of [math]x^n[/math], [math]A^n[/math] for square matrix [math]A[/math], and so on)

Intermediate:
  • Solving systems of linear modular congruences - Chinese Remainder Theorem
  • Solving linear recurrences by fast exponentiation of matrices (e.g., Fibonacci Sequence)
  • Euler's phi function and fast computations
  • MillerRabin primality test

Advanced:
  • Primitive root modulo [math]N[/math]
  • Discrete logarithm
  • Discrete square root
  • Multiplicative function and Mbius inversion Formula
  • Farey sequence and applications
  • (And there should be a lot more)

Quite often, a single problem will require some of the following items together with Number Theory knowledge, so that you can solve it:
  • Dynamic Programming
  • (Basic) Enumerative Combinatorics: [math]_nC_r[/math] and [math]_nP_r[/math]
  • Lucas' Theorem
  • Inclusion-exclusion principle
  • Probability

Finally, feel free to correct errors and add any missing important items. Any suggestion is welcome. :D

Conway and Guy's The Book of Numbers is not as rigorous as some textbooks on number theory.  Instead, it explores key topics in number theory by appealing to the reader's intuition.  The development of various topics lends to understanding of how various properties of numbers arise, which can ease the process of developing algorithms to compute and/or use them. This can also be very useful for software engineering in that some students and/or software engineers struggle with various proofs or derivations of algorithm correctness, runtimes, etc.  IMO, it's a good supplement for textbooks on algorithms that assume the reader understands the number theory that is used to arrive at results.

I assume you are asking for "must-know" knowledge for algorithm programming contests (e.g., the ACM-ICPC, Topcoder SRMs, ...). I'm not so sure if every programmer should know some number theory knowledge.

I participated in the ACM-ICPC for 4 years, entering the World Finals twice. I am not really very strong at Number Theory problems, but I wish to answer as far as I could. The following items are categorized based on subjective judgement.

Basics:
  • LCM and GCD
  • (Extended) Euclidean algorithm
  • Modular arithmetic (addition, subtraction, multiplication)
  • Modular multiplicative inverse ("division") - Existence and computation
  • (Fast) Prime list generation
  • Fast prime factorization (with/without pre-process)
  • Exponentiation by squaring (e.g., computation of [math]x^n[/math], [math]A^n[/math] for square matrix [math]A[/math], and so on)

Intermediate:
  • Solving systems of linear modular congruences - Chinese Remainder Theorem
  • Solving linear recurrences by fast exponentiation of matrices (e.g., Fibonacci Sequence)
  • Euler's phi function and fast computations
  • MillerRabin primality test

Advanced:
  • Primitive root modulo [math]N[/math]
  • Discrete logarithm
  • Discrete square root
  • Multiplicative function and Mbius inversion Formula
  • Farey sequence and applications
  • (And there should be a lot more)

Quite often, a single problem will require some of the following items together with Number Theory knowledge, so that you can solve it:
  • Dynamic Programming
  • (Basic) Enumerative Combinatorics: [math]_nC_r[/math] and [math]_nP_r[/math]
  • Lucas' Theorem
  • Inclusion-exclusion principle
  • Probability

Finally, feel free to correct errors and add any missing important items. Any suggestion is welcome. :D

Conway and Guy's The Book of Numbers is not as rigorous as some textbooks on number theory.  Instead, it explores key topics in number theory by appealing to the reader's intuition.  The development of various topics lends to understanding of how various properties of numbers arise, which can ease the process of developing algorithms to compute and/or use them. This can also be very useful for software engineering in that some students and/or software engineers struggle with various proofs or derivations of algorithm correctness, runtimes, etc.  IMO, it's a good supplement for textbooks on algorithms that assume the reader understands the number theory that is used to arrive at results.
